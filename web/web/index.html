Introduction top

P2P networks can reach huge dimensions such as hundreds of thousands of nodes and those elements can join and leave (gracefully or crashing) the system continously. These P2P system properties are very challenging to deal with. Evaluating a protocol in a such environment, expecially in its early stages of development, it is not  feasible.  On the Internet are available distributed planetary-scale open platforms (e.g.: PlanetLab) to develop and deploy network services, but these solutions don't include more than about 200 nodes: really too small compared to available P2P applications. Thus, for large-scale systems, a simulation testbed is mandatory.

Peersim has been developed with extreme scalability and support for dynamicity in mind; it should be considered as a framework composed by many simple extendable and pluggable components. To reach these goals, some simplifying assumpions have been made (such as throw away all the details about the transport communication protocol stack). Peersim is closely related with BISON project and it is a full open source software.
All the the code is written in the Java language.
